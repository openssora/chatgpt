2026年国内如何安装使用 Grok Build 教程！附 SuperGrok 的升级订阅教程。

最近 X AI 也就是 Grok，马斯克家的，也开始进击编码圈了。上个月大家还在卷 Claude Code、 Codex，这个月 xAI 就推出了自家的 Grok Build。

我用了一天，整体编码能力还是输 Claude Code 和 Codex，（ 我的工作场景更喜欢现在的 Codex ），今天写一篇 Grok Build 安装使用教程分享给大家。

一、Grok Build 是什么

xAI 官方推出的终端 AI 编码代理（Coding Agent CLI），2026 年 5 月进入早期 Beta。

和 GitHub Copilot 那种代码补全插件不一样，它更像是一个住在终端里的工程师。你告诉它要做什么，它会自己规划步骤、读项目代码、写新功能、跑测试，最后还能帮你 git commit。

几个比较有意思的特性：

  • 全屏交互式 TUI 界面，支持鼠标点击
  • Plan Mode 规划模式，先出方案再动手
  • 大任务会自动拆给多个子代理并行处理
  • 支持无头模式，方便塞进脚本和 CI/CD
  • 支持 ACP 协议，能接入 IDE 和其他 Agent 框架

定位上和 Claude Code、Aider 这类工具是直接竞争关系。

二、使用前的准备

订阅方面，需要 SuperGrok 或 X Premium Plus 会员。还没订阅的 SuperGrok 的伙伴，或者不知道怎么订阅的，可以使用 SuperGrok 一键升级系统（grokng.com）。

比如：https://grokng.com 内购升级比较优惠。

如果你需要制作 AI 视频之类的， Grok 目前生成速度挺不错。

系统支持 macOS、Linux、Windows 三个平台。我自己在 Mac 上用得最多，体验也最完整。Windows 上也能跑，但偶尔会遇到一些小问题。

另外建议在 Git 仓库的根目录下运行，它会自动读取仓库结构和配置文件，效果好不少。

刚上手的话，先 /help 看一眼就行。

六、Plan Mode 值得单独说一下

这个功能我觉得是 Grok Build 最值得用的设计。

之前用一些 AI 工具最头疼的就是：你让它改个东西，它二话不说就上手，结果代码改乱了、依赖装错了、测试也跑不过。回滚都麻烦。

Plan Mode 解决的就是这个问题。开启之后，它会先输出一份完整的执行计划，告诉你打算做哪几步、改哪些文件、用什么方案。你可以：

  • 直接批准
  • 评论修改某一步
  • 让它重新规划

确认之后才会动手，每个文件改动也会以 diff 形式展示。

进入 Plan Mode 有三种方式。

第一种是快捷键，在界面里按 Shift + Tab 循环切换，看到状态栏显示 Plan 就行：

十、一些使用上的小经验

用了一周，总结几点：

在 Git 仓库根目录下运行效果最好，它能读到完整的上下文。

大项目尽量先用 Plan Mode 过一遍方案，别一次性让它大改。

支持图片粘贴。我现在调 UI 经常截图丢进去让它看，比文字描述高效。

可以同时开几个实例，一个改前端、一个写文档，互不干扰。

现在还是 Beta，偶尔有些小 bug，遇到了用 /feedback 提就行，官方反馈速度还算快。

最后

我自己用下来的感觉是，Grok Build 和 Claude Code 各有优劣，前者在 Plan Mode 和并行子代理这两点上做得更细，后者在代码质量上还是有些优势。如果你已经是 SuperGrok 会员，可以尝试一下。
